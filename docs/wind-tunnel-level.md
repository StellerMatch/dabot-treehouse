# Wind Tunnel Level

Status: prepared candidate level / internal-only setup
Date: 2026-06-09

## Plain Meaning

Wind Tunnel is an optional Treehouse level for external processed scenarios.

When a project qualifies, Treehouse sends a sanitized idea packet into an outside company/tool/system, watches what it produces, then brings the result back into Treehouse as comparison material.

Wind Tunnel is not the builder, judge, or final product direction. It is an outside pressure test.

## Placement

Preferred placement:

```text
Canopy Level -> Wind Tunnel Level -> Branchworks Level
```

Reason:

- Canopy should clarify the project enough that an outside tool can understand it.
- Wind Tunnel can then produce an external scenario before Treehouse-native experiment work begins.
- Branchworks, Tinker, Momma, and the Build-A-Bears can use the external result as challenge/comparison material.

## Qualification

A project qualifies only when:

- the idea is clear enough to sanitize and send outside;
- the outside run would reveal useful assumptions, prototype directions, workflow shape, or feature gaps;
- no private/sensitive data, credentials, accounts, customer data, or protected internal files are needed;
- Boss/Chief approves the tool/run if subscription credits, account access, or external data transfer is involved;
- Treehouse has a clear reason to compare the external result before Branchworks.

## Output Packet

Each Wind Tunnel run should create a `wind_tunnel_scenario_packet` with:

- qualification decision and reason;
- sanitized idea packet sent out;
- tool/company used;
- subscription/credit note;
- raw or summarized outside output;
- features and assumptions the outside system introduced;
- missing pieces or shallow parts;
- risks, off-target areas, or rejected suggestions;
- useful pieces to feed into Treehouse;
- comparison prompts for Tinker, Momma, and the Build-A-Bears.

## Boundaries

Wind Tunnel does not approve:

- signup/login;
- new subscriptions;
- spending or credit use beyond an approved run;
- private data sharing;
- external account access;
- deployment;
- publishing;
- automation activation;
- Lovable submission;
- final product direction;
- route/config/runtime changes;
- bot authority changes.

## Relationship To Baby Bears

Wind Tunnel is an outside scenario generator.

The Build-A-Bears are Treehouse-native creative builders:

- Ace creates clean-structure directions.
- Bolt creates bold creative directions.
- Craft creates polished usable directions.
- Momma coordinates and packages the blind-build process.

Wind Tunnel can feed or challenge Bear work. It does not replace it.
