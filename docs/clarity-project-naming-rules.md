# Clarity Project Naming Rules

Clarity should name the project from the concrete thing plus the useful action,
not from random words in the intake.

## Naming Rule

1. Find the main subject.
   - Examples: lawn mowing, robotic mower, appointment booking, meal planning,
     client intake, recipe shelf, pet care.
2. Find the main function.
   - Examples: scheduler, tracker, planner, booking app, booking service,
     dispatch tool, marketplace, builder, journal, dashboard.
3. Combine them into a readable working title.
   - Keep the title 2-5 words.
   - Prefer concrete words over vague words.
   - Remove filler such as where, own, my, we, our, and new unless the word is
     truly part of the product.
   - Do not use poetic, random, or brand-like word combinations.
   - Treat the result as an editable working title, not final branding.

The fallback question is:

> What is this mainly about, and what does it help the user do?

## Examples

- Robotic Lawn Scheduler
- Mower Booking Service
- Lawn Robot Dispatch
- Client Intake Tracker
- Recipe Shelf App
- Pet Care Planner

## Product Behavior

When the app generates or cleans a project title, it should use this rule for
front-door idea capture, Library saved ideas, and Dashboard title cleanup. The
same intake should not receive different titles just because it moved to a
different screen.
